基础指令😊

help：帮助命令

i：info,查看信息(输入info可查看所有子命令)

i b：查看所有断点信息（编号、地址）

i r：查看寄存器

i f：查看函数名（需保留符号）

show：查看调试器自身信息

show args：查看程序参数

rdi：直接打印寄存器值（如 $rdi）

backtrace：查看调用栈

q：退出 gdb

vmmap：查看内存布局

执行指令(好理解,但很关键,最好动手看看😊)

s：源码层面的单步步入（step into）

si：汇编层面的单步步入

n：源码层面的步过（step over）

ni：汇编层面的步过

c：继续执行直到断点

r：重新运行

start：运行到main起始处

断点操作(重要!)

下断点（break）

b *0x123456：给某地址下断点(最常用)

b *$rebase(0x123456)：PIE位置无关地址断点

b fun_name：给函数下断点

b file.c:15：给某源码行下断点

b +0x10：当前停住位置偏移下断点

break fun if $rdi==5：条件断点

删除、禁用断点

info break：查看断点编号

delete 5：删除5号断点

disable 5：禁用

enable 5：启用

clear：清除当前文件所有断点

内存断点（watch）

watch 0x123456：该地址值改变时断

watch a：变量改变时断

info watchpoints：查看watch列表

捕获断点（catch）

catch syscall：系统调用断住

tcatch syscall：只断一次

其他：throw / catch / exec / fork / vfork / load / unload / syscall 等

查看内存(x 指令)也很重要!

分析内存布局时必备哦~

格式：x /nfu addr

n：显示几个单元

u：每个单元大小（b/h/w/g）

f：显示格式（x/u/d/t/a/c/f/s/i）

示例：

x /10gx 0x123456：十个8字节单元

x /10xd $rdi：打印rdi指向的十个4B数值

x /10i 0x123456：打印指令

打印指令（p）

p fun_name：打印函数地址

p 0x10-0x08：计算表达式

p &a：查看变量地址

p *(0x123456)：查看地址值

p $rdi：打印寄存器

修改/查找指令

set $rdi=0x10：改寄存器

set *(0x123456)=0x10：改内存

search rdi：查指令

find “hello”：找字符串(pwndbg特有)

堆调试(pwndbg独有)我也不会,先忽略😁😁

arena：查看主 arena

arenas：所有 arena

arenainfo：更美观显示

bins：最常用,查看所有 bins

fastbins

largebins

smallbins

unsortedbin

tcache

tcachebins

heap：显示所有堆块

heapbase：堆基址

parseheap：更清晰的堆结构

tracemalloc：跟踪堆操作

其他pwndbg常用指令

cyclic 50：生成50字节的溢出模式(测试找偏移量用)

$rebase：自动处理PIE偏移

codebase：打印PIE偏移(本地😅)

stack：查看栈

retaddr：定位返回地址

canary：直接看canary(仅本地有用,而且每次都会变😅)

plt：查看PLT

got：查看GOT

hexdump：ida风格hex显示
